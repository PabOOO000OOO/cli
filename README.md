Предметная область:
Пиццерия

Сущности:
-Товары (пицца, напитки, десерты) - название, описание, цена, категория
-Категории товаров (пицца, напитки, десерты)
-Пользователи - регистрация, авторизация, роли
-Роли пользователей (администратор, пользователь)
-Корзина покупок - связь пользователя с товарами и количеством

Что вынесено в каждый слой и почему

domain — сами данные (классы-модели) и правила их проверки. Этот слой ничего
не знает о базе данных и о консоли.
-Почему: позволяет легко менять БД или интерфейс, не затрагивая бизнес-логикy

data — работа с базой данных (SQLite). Здесь мы сохраняем и загружаем данные.
-Почему: изолирует работу с БД, позволяет заменить SQLite на JSON или другую БД

cli (presentation) — общение с пользователем: меню, ввод, вывод.
-Почему: отделяет пользовательский интерфейс от бизнес-логики



Валидации

Реализовано 2 типа валидации:

1 Обязательное текстовое поле (не пустая строка после trim) validateNotEmpty()
На что: Имя пользователя, название товара, телефон и т.д.

2 Числовое поле > 0 validatePositiveInt(), validatePositiveDouble()
На что: Цена товара (double > 0) и Количество в корзине (int > 0)
